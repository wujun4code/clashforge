# 常见问题

## 关于适用场景

### 我是独立开发者 / 程序员，ClashForge 适合我吗？

非常适合。你每天用的工具——GitHub、npm、Claude Code、OpenAI API、Google Search——都对出口 IP 有隐性要求。机场共享 IP 经常触发 reCAPTCHA、API 调用失败、账号风控等问题。

ClashForge 的优势在于：

1. 路由器统一管理，工作电脑不需要单独配置代理软件
2. 支持接入 Cloudflare Worker 自建节点，给工作流量提供干净独享的出口
3. 其他设备（手机、电视）继续用机场，互不干扰

---

### 我做自媒体 / 跨境电商，IP 质量对我很重要，ClashForge 能帮到我吗？

能，尤其是 Cloudflare Worker 节点功能。

平台账号管理（YouTube Studio、Twitter/X、Facebook Ads、Google Ads）对 IP 的一致性和信誉要求很高。机场共享 IP 上可能早就有人在这些平台上做了不好的事，导致整段 IP 被风控。

用 ClashForge 接入自建 Worker 节点，你的账号操作始终来自同一个、干净的 Cloudflare IP，降低被误判的概率。

---

### 我只想在一台电脑上用代理，还需要 ClashForge 吗？

不一定需要。

ClashForge 的价值主要体现在"全屋多设备统一管理"和"路由器层代理"。如果你只需要在一台电脑上临时用代理，直接在电脑上安装 Clash/Mihomo 客户端更简单。

ClashForge 更适合：有多台设备需要使用代理，或者需要对 IP 质量进行精细管理的场景。

---

## 关于机场订阅

### 我已经有机场，还需要其他节点吗？

取决于你的使用场景：

- **只是日常浏览和娱乐**：机场够了
- **需要稳定访问 OpenAI、Claude，且预算有限**：叠加 Cloudflare Worker 节点（免费），CF IP 信誉比机场好，能减少 reCAPTCHA 和风控
- **外贸收款、广告账号、需要固定 IP**：必须上 VPS 节点，Cloudflare Worker 的 IP 不固定，不适合这类场景

三者可以同时配置，ClashForge 按设备和规则自动分流。

---

### 为什么我用了代理，Google 还是一直要我点 reCAPTCHA？

这是机场共享 IP 的典型问题，不是 ClashForge 的问题，也不是你的操作问题。

你使用的出口 IP 被数百人共用，Google 的系统检测到这个 IP 的请求频率异常，因此对所有从这个 IP 来的请求都触发验证。

解决方法：

1. 切换到其他节点试试
2. 长期解决：配置 Cloudflare Worker 自建节点，使用只有你在用的 IP

---

### 为什么 OpenAI 说我的账号有异常，但我没做任何违规操作？

这是出口 IP 信誉问题。OpenAI 的风控系统会对来自已知"代理/VPN IP"的请求或登录行为进行额外审查。机场的共享 IP 在各大平台的 IP 信誉数据库里通常评分很低，触发风控是概率性的。

建议：

1. 先换节点试试
2. 后续使用 Cloudflare Worker 节点作为 OpenAI 相关服务的出口
3. 如果账号已被限制，联系 OpenAI 客服说明情况

---

### 机场节点突然全部不能用了，怎么办？

先检查：

1. 机场账号是否过期或欠费（登录机场后台查看）
2. 机场是否有维护公告（查看机场官网或 Telegram 频道）
3. 在 ClashForge 里手动更新订阅（获取最新节点列表）

如果机场服务本身出问题，ClashForge 无法帮你修复节点——这是机场服务商的问题。建议：

- 平时准备一个备用机场
- 同时配置 Cloudflare Worker 节点，作为机场不可用时的备选

---

## 关于 Cloudflare Worker 节点

### Cloudflare Worker 节点是什么意思，听起来很复杂？

不复杂。简单理解：

你在 Cloudflare（全球最大的 CDN 服务商）的账号上，创建了一个小程序（Worker），这个程序负责把你的网络请求转发出去。出口 IP 是 Cloudflare 边缘网络的 IP——Cloudflare 的 IP 段在各大平台信誉评分高，不在常规 VPN/代理 IP 黑名单里，所以比机场 IP 更少触发风控。

注意：Cloudflare Worker 的 IP **不是固定的，也不是独享的**。如果需要同一个固定 IP 长期使用（支付账号、广告账号等），请用 VPS 节点。

ClashForge 的 Web 界面帮你自动完成 Worker 的创建和配置，你不需要手写代码。

---

### Cloudflare Worker 节点是免费的吗？

免费账号每天有 **10 万次请求额度**，对正常工作使用基本够用。

10 万次请求足够你打开几千个网页，或者调用 API 几万次。如果你在跑批量脚本或爬虫，可能需要升级到付费套餐（约 $5/月）。

---

### Cloudflare Worker 节点的速度怎么样？

Cloudflare 在全球有 300+ 个边缘节点，Worker 会自动路由到离你最近的节点。对于大陆用户，通常是香港、东京或新加坡节点，延迟在 50-150ms 左右。

速度满足日常工作需求（浏览、API 调用、Claude Code）。如果对速度要求很高（比如大文件下载、4K 视频流媒体），机场的专线节点可能更快。

---

### 如果 Cloudflare 免费额度用完了怎么办？

当天额度用完后，Worker 会暂停响应，直到次日 UTC 00:00 重置。

解决方案：

1. 升级 Cloudflare Workers Paid（$5/月，每月 1000 万次请求）
2. 或者临时切换到机场节点，第二天额度重置后再切回

---

## 关于安装和配置

### ClashForge 是代理服务吗？它能帮我买到节点吗？

不是，ClashForge 不是代理服务商，也不提供节点。

它是一个**管理工具**，安装在你的路由器上，帮你管理已有的代理来源（机场订阅或自建 Worker 节点）。你需要自己准备代理来源。

---

### Web 管理页面在哪里？

```
http://<你的路由器地址>:7777
```

例如路由器地址是 `192.168.20.1`，就访问：

```
http://192.168.20.1:7777
```

不知道路由器地址？Windows 上运行 `ipconfig`，看"默认网关"那一行。

---

### 我需要在每台设备上安装客户端软件吗？

不需要。

ClashForge 安装在路由器上，连接到这个路由器的设备按配置的规则自动使用代理，不需要在每台设备上单独安装任何东西。

这是路由器级代理方案的核心优势。

---

### 打开代理后家里网络全断了怎么办？

立即运行：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 stop
```

或者 SSH 登录路由器后运行：

```sh
clashforgectl stop
```

这会关闭代理相关设置，网络通常在 15 秒内恢复。

恢复后，参考 [不好用怎么办](/guide/troubleshooting#问题四打开开关后全家网络异常) 排查原因。

---

### `upgrade` 和 `deploy` 有什么区别？

| 命令 | 适合谁 | 做什么 |
| --- | --- | --- |
| `upgrade` | 普通用户 | 从 GitHub 下载最新编译好的安装包，安装到路由器 |
| `deploy` | 开发者 | 在本地重新构建前端和后端，再打包安装 |

普通用户只用 `upgrade`。

---

### 诊断报告可以直接发出去吗？

建议只发**脱敏版**：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 diag -Fetch -Redact
```

原始报告可能包含订阅链接（含认证 Token）、内网 IP、账号信息等敏感内容，不要直接发到公开的群组或论坛。
