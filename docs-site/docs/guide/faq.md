# FAQ

## ClashForge 是代理服务吗？

不是。

ClashForge 不卖节点，也不提供代理线路。  
它是一个安装在 OpenWrt 路由器上的管理工具，帮你把已有的代理订阅或配置更方便地用起来。

## 我已经有订阅了，ClashForge 能帮我做什么？

它可以帮你：

1. 把订阅放到路由器上。
2. 用网页启动和停止代理服务。
3. 切换节点、更新订阅。
4. 让连接到路由器的设备更方便地使用代理。
5. 出问题时恢复网络并收集诊断信息。

## 我还没有代理订阅，可以用吗？

不能直接使用代理能力。  
你需要先准备可用的订阅链接或 Clash 配置文件。

## 我需要在每台设备上安装客户端吗？

目标是不需要每台都装。  
ClashForge 把代理能力放到路由器上，设备连接到这个路由器后，就可以按你的设置使用。

第一次建议只让一台设备测试，稳定后再给更多设备用。

## Web 管理页面在哪里？

```text
http://<路由器地址>:7777
```

例如：

```text
http://192.168.20.1:7777
```

## 我该用哪个安装命令？

大多数用户使用：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 upgrade
```

把 `192.168.20.1` 换成你的路由器地址。

## `deploy` 是什么？我要用吗？

普通用户不需要。

`deploy` 是给开发者用的，会重新构建源码并安装到路由器。你只是想使用 ClashForge 的话，用 `upgrade` 就好。

## 打开代理后网络异常怎么办？

先恢复网络：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 stop
```

恢复后再逐步排查，不要连续改很多设置。

## 怎么确认已经生效？

看三件事：

1. Web 页面显示服务正常。
2. 需要代理的网站能打开。
3. 出口 IP 符合你选择的节点。

也可以运行：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 check
```

## 诊断报告能直接发出去吗？

建议只发脱敏报告：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 diag -Fetch -Redact
```

原始报告可能包含订阅链接、Token、内网地址等敏感信息。
