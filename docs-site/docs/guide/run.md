# 启动接管与设备生效

导入来源后，还需要决定两件事：mihomo 内核什么时候启动，以及哪些设备的流量要进入 mihomo。

## 三层开关

ClashForge 把运行状态拆成三层，这样第一次上线更安全。

| 层级 | 做什么 | 建议 |
| --- | --- | --- |
| 管理服务 | Web UI 和 API，端口默认 `7777` | 安装后自动启动 |
| mihomo 内核 | 加载最终 YAML，提供代理端口和 API | 导入配置后启动 |
| 透明代理/DNS 接管 | 让 LAN 设备流量自动进入 mihomo | 验证节点后再开启 |

不要把“Web UI 能打开”等同于“全网已经走代理”。Web UI 只是管理服务，真正影响设备上网的是 mihomo 内核和接管规则。

## 1. 确认管理服务运行

浏览器打开：

```text
http://192.168.20.1:7777
```

远程查看：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 status
```

路由器服务命令：

```sh
/etc/init.d/clashforge status
/etc/init.d/clashforge restart
```

## 2. 启动 mihomo 内核

在 Web UI 中：

1. 进入 Setup 或“配置管理”。
2. 选择可用配置来源。
3. 查看最终配置预览。
4. 点击启动。
5. 回到“概览”，确认内核 PID、运行时长、CPU、内存正常。

启动后先验证：

| 检查项 | 期望结果 |
| --- | --- |
| 代理组 | 能看到 Selector、URLTest、Fallback 等组 |
| 节点测速 | 至少一个节点延迟可用 |
| 运行中配置 | 能看到最终写入的 YAML |
| 日志 | 没有持续的 DNS、端口或 YAML 错误 |

## 3. 先测试节点，不急着全网接管

在“概览”里进行：

1. 点击路由器侧检测。
2. 点击浏览器侧检测。
3. 对比两个结果的出口 IP。
4. 切换一个 Selector 节点，再重新检测。
5. 用域名测试检查目标网站。

这一步的价值是先判断“节点和配置本身是否可用”。如果这里失败，就不要继续开启全局接管。

## 4. 让单台设备生效

建议先选一台不影响工作的测试设备。

准备：

| 准备项 | 说明 |
| --- | --- |
| 设备连接到 OpenWrt | Wi-Fi 或有线都可以 |
| 记录设备 IP | 可在 ClashForge 在线设备或路由器 DHCP 页面查看 |
| 最好绑定静态租约 | 避免设备 IP 变化导致规则失效 |

做法：

1. 在“设备分流”里创建测试设备组。
2. 把该设备 IP 加入设备组。
3. 给它配置一个明确出口，例如某个 VPS 或 Worker 节点。
4. 保存并重新生成配置。
5. 启动内核，开启接管。
6. 在该设备上查询出口 IP。

## 5. 开启透明代理

**这个功能是干嘛的？**
透明代理让设备不用手动设置 HTTP/SOCKS 代理，路由器在防火墙层把符合条件的流量送入 mihomo。

**解决什么问题？**
手机、电视、平板、IoT 设备往往不好配置代理。透明代理让这些设备也能按路由器策略生效。

**如何使用？**

1. 确认 mihomo 内核已启动。
2. 确认至少一个节点可用。
3. 进入“高级管理”或 Setup 的网络设置。
4. 选择透明代理模式。
5. 保持绕过 LAN 启用。
6. 应用接管。
7. 测试一台设备。

**实现效果和价值**

| 效果 | 价值 |
| --- | --- |
| 设备免配置 | 不需要每台设备安装客户端 |
| 分流集中控制 | 策略由路由器统一执行 |
| 可停止恢复 | 出问题可用 `stop` 清理规则 |

## 6. 开启 DNS 接管

**这个功能是干嘛的？**
DNS 接管让设备的域名解析进入 ClashForge/mihomo 管理，减少 DNS 泄漏和规则错配。

**解决什么问题？**
如果流量走代理但 DNS 直连，可能出现目标域名解析到错误 IP、规则命中异常、海外服务打不开。

**如何使用？**

1. 先确认透明代理或核心代理已经可用。
2. 进入“高级管理”。
3. 开启 DNS 功能。
4. 选择 fake-ip 或 redir-host。
5. 选择 dnsmasq 共存方式。
6. 应用后测试国内和海外域名。

**实现效果和价值**

| 效果 | 价值 |
| --- | --- |
| DNS 与规则协同 | 分流更可预测 |
| 降低 DNS 泄漏 | 海外域名不再被本地 DNS 干扰 |
| 可逐步启用 | 先内核，再透明代理，再 DNS |

## 7. 何时开启开机自动接管

建议满足这些条件后再开启：

| 条件 | 为什么 |
| --- | --- |
| 连续几天使用稳定 | 说明配置和节点可靠 |
| 已知道如何执行 `stop` | 出问题能恢复 |
| 关键设备已绑定静态 IP | 设备分流不会漂移 |
| 有备用节点或备用订阅 | 单节点故障不影响全网 |

## 8. 停止接管和恢复网络

Windows：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 stop
```

macOS / Linux：

```sh
./scripts/clashforgectl --router 192.168.20.1 stop
```

路由器本机：

```sh
cd /tmp
wget -O clashforgectl.sh https://raw.githubusercontent.com/wujun4code/clashforge/main/scripts/clashforgectl.sh
sh clashforgectl.sh stop
```

`stop` 会尽量清理：

| 清理对象 | 目的 |
| --- | --- |
| clashforge/mihomo 进程 | 停止代理服务 |
| nftables/iptables 规则 | 退出透明代理 |
| 策略路由 | 恢复普通路由 |
| DNS 接管配置 | 恢复 dnsmasq |
