# 让设备开始使用

添加订阅并启动服务后，还需要决定：哪些设备要通过路由器使用代理。

建议第一次只让一台设备测试，确认没问题后再扩大到更多设备。

## 推荐顺序

1. 启动 ClashForge 代理服务。
2. 在 Web 页面确认节点可用。
3. 让一台手机或电脑开始使用代理。
4. 打开几个需要代理的网站测试。
5. 没问题后，再让更多设备使用。

## 启动代理服务

在浏览器打开：

```text
http://192.168.20.1:7777
```

在页面里点击启动。启动后你应该能看到运行状态变化。

也可以用命令确认：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 status
```

## 让设备通过路由器使用代理

在 Web 页面里打开相关开关后，路由器会尝试让连接到它的设备使用代理服务。

普通用户可以这样理解：

1. 你不用在每台设备上都安装代理软件。
2. 设备连接到这个路由器。
3. 路由器帮你把需要代理的访问转给代理服务。

第一次打开后，先用一台设备测试。

## 怎么测试

用这台设备打开：

1. 平时需要代理才能访问的网站。
2. 平时不需要代理的国内网站。
3. 一个查看出口 IP 的网站。

期望结果：

1. 需要代理的网站能打开。
2. 普通网站仍然正常。
3. 出口 IP 符合你的代理节点位置。

## 不建议第一次就打开所有高级选项

有些设置会影响所有设备的上网体验。第一次建议保持默认，先确认能用。

等你熟悉后，再考虑：

1. 哪些设备必须走代理。
2. 哪些设备永远不走代理。
3. 哪些网站直接访问。
4. 哪些网站通过代理访问。

## 出问题怎么恢复

如果打开后设备无法上网，先运行：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 stop
```

路由器本机也可以：

```sh
clashforgectl stop
```

先恢复网络，再看 [不好用怎么办](/guide/troubleshooting)。
