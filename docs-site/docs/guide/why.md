# 产品定位与适用人群

ClashForge 是安装在 OpenWrt 路由器上的 mihomo 管理控制台。它不是机场、不是代理节点销售平台，也不是另一个代理内核；它负责把订阅、节点、DNS、透明代理、设备分流、订阅发布和诊断恢复放进一个统一的 Web 管理界面。

如果你只是在一台电脑上偶尔开代理，桌面客户端已经够用。如果你有多台设备、多人协作、固定业务出口、海外 AI 工具或跨境业务，ClashForge 的价值才会真正显现。

## 一句话理解

ClashForge 把“每台设备自己折腾代理”变成“路由器统一管理出口策略”。

它关注四件事：

| 方向 | ClashForge 做什么 |
| --- | --- |
| 出口质量 | 把机场、自建 VPS、Cloudflare Worker 节点放在一起管理 |
| 设备策略 | 按设备、业务和场景决定走哪个出口 |
| DNS 与规则 | 让透明代理、DNS 接管、规则集和 GeoData 可视化 |
| 运维恢复 | 网络异常时能快速停止接管、恢复上网、生成诊断报告 |

## 适合哪些用户

### 1. 电商独立站与跨境运营团队

你可能每天要访问 Shopify、Stripe、PayPal、Google Ads、Meta Ads、Google Analytics、Gmail、Cloudflare、TikTok Shop、Amazon Seller 等后台。

ClashForge 能解决的问题：

| 问题 | 为什么常见 | ClashForge 的做法 |
| --- | --- | --- |
| 后台频繁验证 | 共享机场 IP 信誉不稳定 | 关键工作电脑绑定固定 VPS 出口 |
| 支付或广告后台敏感 | 平台会关注登录来源一致性 | 按设备分流，业务设备长期走同一出口 |
| 团队设备配置混乱 | 每个人客户端设置不同 | 路由器统一策略，减少人为差异 |
| 国内服务变慢 | 全局代理误伤国内流量 | 规则集、GeoData 和直连策略统一管理 |

推荐策略：

| 设备或场景 | 推荐出口 |
| --- | --- |
| 运营主力电脑 | VPS/SSH 固定节点 |
| 财务和支付操作设备 | VPS/SSH 固定节点，尽量少切换 |
| 素材下载、普通浏览 | 机场订阅或 Worker 节点 |
| 国内 ERP、微信、淘宝、银行 | DIRECT 直连 |

### 2. TikTok 电商、自媒体与内容团队

你可能需要访问 TikTok、YouTube Studio、X、Instagram、Canva、CapCut、Google Drive、Creator Marketplace、广告后台和素材平台。

ClashForge 能解决的问题：

| 问题 | 为什么常见 | ClashForge 的做法 |
| --- | --- | --- |
| 创作者后台登录反复验证 | 出口 IP 变化频繁或信誉低 | 给运营设备配置稳定出口 |
| 上传、剪辑、素材工具分散 | 手机、电脑、平板都要访问海外服务 | 路由器层统一接入，不用每台设备装客户端 |
| 团队成员误用节点 | 客户端随意切换导致来源不一致 | 设备分流限制关键设备可选节点 |
| 直播或内容制作需要备用线路 | 单一节点故障会影响工作 | 同时保留机场、Worker、VPS 多类来源 |

推荐策略：

| 设备或场景 | 推荐出口 |
| --- | --- |
| 账号运营电脑 | 稳定 VPS 节点 |
| 素材浏览与普通工具 | Worker 或机场节点 |
| 电视、手机娱乐 | 机场节点 |
| 国内协作工具 | DIRECT 直连 |

### 3. 需要大量访问海外 AI 资源的个人或团队

你可能每天使用 OpenAI、Claude、Gemini、Perplexity、GitHub、npm、Docker Hub、Hugging Face、Replicate、Cursor、Claude Code 或 OpenAI API。

ClashForge 能解决的问题：

| 问题 | 为什么常见 | ClashForge 的做法 |
| --- | --- | --- |
| OpenAI/Claude 访问不稳定 | 节点 IP 被频繁风控 | 给 AI 工作设备绑定更干净的出口 |
| API 或开发工具偶发失败 | DNS、规则、节点质量混在一起 | 通过路由器侧和浏览器侧检测定位问题 |
| 多台设备都要用 AI 工具 | 每台机器配置代理成本高 | 路由器统一接管 |
| GitHub/npm 与国内访问冲突 | 全局代理或全局直连都不理想 | 规则集与 GeoData 管理让分流更可控 |

推荐策略：

| 设备或场景 | 推荐出口 |
| --- | --- |
| 主力开发电脑 | VPS 固定节点或质量较好的 Worker |
| API 批处理任务 | 独立 VPS 节点，避免与日常流量混用 |
| 手机、平板访问 AI 工具 | Worker 或机场节点 |
| 国内镜像、内网服务 | DIRECT 直连 |

### 4. 开发者、小工作室与网络管理员

如果你在维护一个家庭、办公室或小团队网络，ClashForge 更像一个轻量网络控制台。

适合你使用的原因：

| 需求 | ClashForge 提供的能力 |
| --- | --- |
| 想知道当前内核是否正常 | 仪表盘展示 PID、资源、连接、流量 |
| 想看最终生成的 mihomo 配置 | 运行中配置只读预览 |
| 想确认域名命中了哪条规则 | 规则集搜索与 rule-provider 同步 |
| 想远程升级和诊断 | Windows/macOS/Linux 远程控制脚本 |
| 想自建团队订阅 | Cloudflare Worker + KV 发布订阅链接 |

### 5. 多设备家庭和轻量办公室

如果家里有电视、手机、平板、电脑、NAS、游戏机，ClashForge 可以把“哪些设备走代理，哪些直连”从每台设备移动到路由器上。

适合的情况：

| 场景 | 价值 |
| --- | --- |
| 家里多人共用订阅 | 统一更新订阅，避免每台设备重复配置 |
| 电视和手机只看流媒体 | 单独分流到娱乐节点 |
| 工作电脑需要稳定出口 | 与娱乐设备隔离 |
| 出问题要快速恢复全家网络 | `stop` 命令清理接管，优先恢复上网 |

## 为什么不能只依赖机场订阅

机场订阅的本质是共享节点。共享节点适合日常浏览、流媒体和低风险访问，但它有几个结构性限制。

| 限制 | 可能表现 |
| --- | --- |
| 出口 IP 被很多人共用 | Google reCAPTCHA、账号验证变多 |
| IP 历史行为不可控 | 某些平台把该 IP 标记为代理或高风险 |
| 节点随服务商调整 | 出口地区、IP、可用性可能变化 |
| 所有设备混用同一出口 | 工作、娱乐、支付、广告流量相互影响 |

ClashForge 不会让共享 IP 变成独享 IP，但它能帮你把不同出口用在正确的地方：普通设备走机场，轻量工作走 Worker，关键业务走 VPS。

## 三类出口怎么选

| 出口类型 | 优点 | 局限 | 适合场景 |
| --- | --- | --- | --- |
| 机场订阅 | 便宜、节点多、速度选择多 | 共享 IP、信誉不可控 | 娱乐、普通浏览、备用线路 |
| Cloudflare Worker | 成本低、Cloudflare 网络信誉好、部署方便 | IP 不固定、不独享、受请求额度限制 | 轻量工作、一般 AI 访问、备用出口 |
| VPS/SSH 节点 | IP 固定且只由你使用 | 需要购买 VPS，需维护 | 支付、广告、账号后台、长期稳定 AI 工作 |

## 推荐的分层模型

一个更稳的网络不是“所有流量都走最快节点”，而是分层。

```text
关键业务设备
  -> 固定 VPS 出口
  -> 适合支付、广告、账号后台、API 批处理

日常工作设备
  -> Worker 或质量较好的订阅节点
  -> 适合 AI 工具、GitHub、资料查询

娱乐设备
  -> 机场订阅
  -> 适合流媒体、普通浏览

国内网站与内网服务
  -> DIRECT
  -> 避免无意义绕路
```

ClashForge 的设备分流、规则集、订阅发布和诊断功能，都是围绕这个模型设计的。

## 你需要准备什么

| 必需程度 | 准备项 | 说明 |
| --- | --- | --- |
| 必需 | OpenWrt 路由器 | 需要能 SSH 登录，且架构受 Release 包支持 |
| 必需 | 一台电脑 | Windows、macOS、Linux 都可以，用于远程安装和维护 |
| 至少一种 | 代理来源 | 机场订阅、Cloudflare Worker、VPS/SSH 节点任选 |
| 推荐 | DHCP 静态租约 | 设备分流依赖源 IP，关键设备建议固定 IP |
| 可选 | Cloudflare 账号 | 用于 Worker 节点和订阅发布 |
| 可选 | 海外 VPS | 用于固定独享出口 |
