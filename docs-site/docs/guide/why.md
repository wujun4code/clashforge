# 为什么需要管好自己的网络出口

在开始安装之前，值得花 5 分钟了解一个关键问题：**你明明有代理，但访问 Google、OpenAI、Claude 还是经常出问题，根本原因是什么？**

绝大多数情况下，根源不在你，在你用的那个出口 IP 地址。

---

## 机场订阅的本质：共享出口 IP

你付费买的机场订阅，本质上是租用了服务商的一批服务器节点。这些服务器的 IP 地址，同时被订阅了这个机场的**几百甚至几千个用户共享使用**。

这带来了一个结构性问题：

| 现象 | 原因 |
| --- | --- |
| Google 经常弹出 reCAPTCHA 验证 | 同一个 IP 的请求量巨大，被识别为可疑流量 |
| OpenAI 提示账号异常 | 这个 IP 上曾有人触发过风控或违规 |
| Claude 登录失败或要求验证 | 这个 IP 段已被标记为高风险代理 IP |
| Stripe / PayPal 要求额外验证 | 支付系统把这个 IP 识别为"代理/VPN IP" |
| 社媒平台账号被限流 | IP 被同段其他用户的行为污染 |

**你的操作完全正常，但你在用一个"脏"的出口 IP。**

---

## IP 信誉：你不能控制的变量

互联网上有专门的 IP 信誉评分系统（例如 Maxmind、IPQualityScore）。这些系统持续追踪每个 IP 地址的历史行为：

- 这个 IP 有没有发送垃圾邮件？
- 这个 IP 有没有暴力破解登录？
- 这个 IP 是否出现在已知 VPN/代理的 IP 列表里？

热门机场的节点 IP，因为被大量用户频繁使用，很容易在这些系统里累积负面记录。哪怕你自己从没做过任何异常操作，**你的 IP 信誉评分也可能很低**，因为你在为其他几百个人的行为背书。

## 三种出口方案的对比

| 方案 | IP 是否独享 | IP 是否固定 | 费用 | 适合场景 |
| --- | --- | --- | --- | --- |
| **机场共享节点** | 否，数百人共用 | 否 | 按月付费 | 日常浏览、看视频、娱乐 |
| **Cloudflare Worker 节点** | 否，但属于 CF 信誉体系 | 否，IP 会变 | 免费（10万次/天） | 预算有限、轻度工作、一般 API 调用 |
| **VPS / SSH 节点** | 是，只有你用 | 是，固定不变 | ≈ $3–10/月 | 外贸支付、账号管理、OpenAI/Claude 稳定使用、需要固定 IP |

### Cloudflare Worker 节点

Cloudflare Worker 是 Cloudflare 提供的边缘计算服务，免费账号每天有 **10 万次请求额度**。

它的优势不是"IP 独享"，而是：**Cloudflare 的 IP 段本身在各大平台的信誉评分高**。机场 IP 通常出现在已知的"VPN/代理 IP"数据库里，Cloudflare 的 IP 不会。所以用 CF Worker 出口，比用普通机场节点触发 reCAPTCHA 和风控的概率要低很多。

但它的局限性同样明显：IP 不是固定的，不是独享的，偶尔也会变。如果你的业务需要**同一个 IP 长期稳定**（例如广告账号、外贸收款、平台账号管理），CF Worker 不够用。

### VPS / SSH 节点

自己购买一台海外 VPS（虚拟私有服务器），在上面搭建代理服务。这台服务器的 IP 地址只有你在使用，固定不变。

优势：
- IP 完全独享，不会因为其他人的行为被污染
- IP 地址固定，适合需要"同一个 IP 长期出现"的业务
- 可以选择对目标业务友好的地区（例如美国 IP 用于 Google Ads，日本 IP 用于日本电商）

劣势：
- 需要花钱（便宜的 VPS 大约 $3–5/月）
- 需要自己搭建代理服务（ClashForge 支持导入 SSH 节点配置）
- IP 是数据中心 IP，部分场景仍可能被识别为"服务器 IP"而非"家庭 IP"

## 推荐的多轨策略

最合理的配置不是非此即彼，而是**按场景分流**：

```
工作设备（主力电脑）
  └── 走 VPS/SSH 节点（固定独享 IP）
      适合：Claude Code、OpenAI API、Google Ads、
            外贸收款（Stripe/PayPal）、社媒账号管理

Cloudflare Worker 节点
  └── 备用/轻量工作出口（零成本）
      适合：一般 API 调用、偶发的工作需求

娱乐设备（手机、iPad、电视）
  └── 走机场节点
      适合：YouTube、流媒体、一般浏览

国内网站（淘宝、微信、B站）
  └── 直连，不走代理
```

ClashForge 在路由器层实现这个分流逻辑，你只需要在 Web 管理页面配置一次，全屋设备自动生效。
