# 不好用怎么办

排查问题时先记住一句话：**先恢复上网，再找原因**。

---

## 第一步：先恢复网络

如果家里设备无法上网，不要先改更多设置，先执行：

Windows：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 stop
```

路由器本机：

```sh
clashforgectl stop
```

执行后等 10-15 秒，检查设备是否能正常上网。网络恢复后再慢慢排查原因。

---

## 问题一：管理页面打不开

先确认地址正确：

```
http://192.168.20.1:7777
```

再检查服务状态：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 status
```

常见原因：

| 现象 | 可能原因 | 处理方式 |
| --- | --- | --- |
| 浏览器提示"无法访问" | 路由器地址不对 | 确认路由器实际 IP，替换 192.168.20.1 |
| 连接超时 | ClashForge 没有启动 | SSH 登录路由器，手动执行 `clashforgectl start` |
| 403 或其他错误 | 端口被占用 | 检查 7777 端口是否有其他程序 |
| 电脑能 ping 通路由器但页面打不开 | 防火墙拦截 | 检查路由器防火墙规则 |

## 问题二：订阅添加后没有节点

可能原因：

1. 订阅链接复制不完整，或复制时带了多余空格
2. 路由器无法访问订阅服务器（网络问题）
3. 订阅链接需要特殊 User-Agent（部分机场要求）
4. 订阅已过期或账号欠费

排查步骤：

1. 在电脑浏览器里打开这个订阅链接，确认能看到 YAML 格式的内容
2. 重新从机场后台复制链接，不要从聊天记录里复制（可能不完整）
3. 在 ClashForge 页面手动点击"更新订阅"

---

## 问题三：节点有，但特定网站打不开

这是最常见的情况，原因可能有几种，按概率从高到低排查：

### 3.1 这个节点 IP 被目标网站封禁

**症状**：换一个节点后能打开，说明是节点 IP 问题，不是你的配置问题。

这就是机场共享 IP 的典型问题。解决方案：

- 短期：多换几个节点，找一个目前没有被封的
- 中期：配置 [Cloudflare Worker 节点](/guide/config#方式二添加-cloudflare-worker-节点)（免费，CF IP 信誉比机场好）
- 长期（需要固定 IP 的场景）：配置 [VPS 节点](/guide/config#方式三添加-vps--ssh-节点)，独享固定 IP

### 3.2 触发了 reCAPTCHA 或验证码

**症状**：Google 搜索要求点图片验证，OpenAI 登录要求短信验证，支付时要求额外验证。

根本原因：你的出口 IP 信誉评分低，被平台标记为可疑来源。

解决方案：

1. 立即：换其他节点试试
2. 中期：改用 Cloudflare Worker 节点（CF IP 信誉好，不在 VPN 黑名单）
3. 根本解决（需要固定 IP 的业务）：改用 VPS 节点，固定独享 IP

关于 IP 信誉问题的详细解释，参考 [为什么需要管好自己的网络出口](/guide/why)。

### 3.3 OpenAI / Claude 提示账号异常

**症状**：登录后提示"你的账号存在异常活动"，或要求绑定手机，或直接封号。

这类问题**不是 ClashForge 的问题**，是出口 IP 的信誉问题：

1. 更换节点后重试
2. 清除浏览器缓存和 Cookies 后重试
3. 如果账号已被标记，联系对应平台客服
4. 后续使用请配置 VPS 节点（固定独享 IP）或至少改用 Cloudflare Worker 节点，避免再次触发

### 3.4 分流规则不匹配

**症状**：某个网站明明需要代理，但流量直连了；或者国内网站被代理了，变慢了。

检查步骤：

1. 在 ClashForge 管理页面查看连接日志，确认目标域名走了哪条规则
2. 如果规则不对，手动更新订阅（新版订阅通常包含更新的规则集）
3. 熟悉 Clash 规则的用户可以添加自定义规则

### 3.5 节点本身不可用

**症状**：所有节点都打不开，换节点也无效。

检查步骤：

1. 确认机场订阅没有过期（在机场后台查看账号有效期）
2. 在机场官网查看是否有公告（服务器维护、IP 更换等）
3. 执行检查命令：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 check
```

## 问题四：打开开关后全家网络异常

先执行恢复命令：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 stop
```

恢复后按这个顺序重新操作：

1. 先只启动代理服务，不要打开"全部设备接管"
2. 只在一台设备上测试
3. 确认这台设备正常后，再逐步让其他设备使用

常见原因：

- DNS 设置冲突（代理 DNS 和路由器 DNS 同时生效）
- 路由表规则冲突
- 路由器内存不足，代理进程异常退出

---

## 问题五：之前好用，突然不行了

机场的情况会随时变化，以下原因经常导致"突然不行"：

| 原因 | 判断方式 | 处理方式 |
| --- | --- | --- |
| 节点 IP 被封 | 换节点后好了 | 切换节点，或联系机场反馈 |
| 机场服务器维护 | 机场公告 | 等待或切换到备用机场 |
| 订阅过期 / 账号欠费 | 机场后台查看 | 续费后更新订阅 |
| GFW 规则更新 | 同期其他人也反映 | 等待机场更新节点或规则 |
| 路由器重启后配置丢失 | 重启后失效 | 检查 ClashForge 开机自启设置 |

---

## 问题六：Cloudflare Worker 节点不可用

如果自建的 Worker 节点连接失败：

1. 登录 Cloudflare Dashboard，确认 Worker 仍然存在且处于 Active 状态
2. 检查 Worker 的请求次数是否已超过免费额度（10 万次/天）
3. 在 ClashForge 页面重新部署 Worker（重新走一遍部署流程）
4. 如果长时间不用，Worker 可能被自动暂停，重新部署即可激活

---

## 需要求助

生成脱敏诊断报告后再提问，能大幅提高解决效率：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 diag -Fetch -Redact
```

脱敏报告会隐藏订阅链接、Token 和内网地址。提交问题时请附上：

1. 你想做什么（访问什么网站 / 使用什么服务）
2. 实际看到的现象（截图比文字描述更有效）
3. 路由器型号和 OpenWrt 版本
4. ClashForge 版本
5. 脱敏诊断报告

**不要公开订阅链接、Token 或账号密码。**
