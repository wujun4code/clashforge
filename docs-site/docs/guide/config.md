# 添加代理来源

ClashForge 支持三种代理来源，可以单独使用，也可以同时配置多个：

| 来源类型 | IP 独享/固定 | 费用 | 适合什么 |
| --- | --- | --- | --- |
| **机场订阅** | 否，共享 | 按月付费 | 日常浏览、娱乐、流媒体 |
| **Cloudflare Worker 节点** | 否，但 CF IP 信誉好 | 免费 | 轻度工作、预算为零的场景 |
| **VPS / SSH 节点** | 是，固定独享 | ≈ $3–10/月 | 外贸业务、账号管理、需要固定 IP |

不清楚该用哪种？先看 [为什么需要管好自己的网络出口](/guide/why)。

---

## 方式一：添加机场订阅

### 你需要准备什么

从机场后台复制 **Clash 格式的订阅链接**。订阅链接通常长这样：

```
https://xxxx.xxxxxx.com/api/v1/client/subscribe?token=xxxxxxxxxxxxxxxx
```

不同机场叫法不同，可能叫"Clash 订阅"、"通用订阅"或"复制链接"。如果机场提供多种格式，选"Clash"或"YAML"格式。

### 添加步骤

打开管理页面 `http://192.168.20.1:7777`，进入订阅 / 配置管理：

1. 点击"添加订阅"
2. 粘贴订阅链接
3. 给订阅起个容易识别的名字，例如"主机场"或"备用机场"
4. 保存，点击"更新订阅"拉取最新节点
5. 选中这个订阅，点击启动

### 注意事项

- 不要把订阅链接截图发到群里，链接包含你的认证 token
- 有备用机场的话也可以一起添加，切换时直接在页面选择
- 订阅链接过期或机场关闭后需要重新配置

## 方式二：添加 Cloudflare Worker 节点

Cloudflare Worker 节点的核心优势是**免费**，以及出口 IP 属于 Cloudflare 的 IP 段——Cloudflare 不在常规的"VPN/代理 IP"黑名单里，信誉比典型机场 IP 好。

但要明确：**Cloudflare Worker 的 IP 不是固定的，也不是独享的**。如果你的业务需要同一个 IP 长期稳定（广告账号、支付账号、平台账号管理等），请用方式三（VPS/SSH 节点）。

### 你需要准备什么

- 一个 Cloudflare 账号（免费注册：cloudflare.com，不需要信用卡）
- 免费额度：**每天 10 万次 Worker 请求**

### 添加步骤

打开管理页面 `http://192.168.20.1:7777`，找到"Cloudflare Worker 节点"或"节点管理"入口：

1. 按引导完成 Cloudflare OAuth 授权
2. 点击"部署 Worker 节点"
3. ClashForge 会自动在你的 Cloudflare 账号下创建并配置 Worker
4. 部署完成（约 30–60 秒）后，节点列表里会出现对应条目
5. 选中该节点，测试连通性

### 适合用 CF Worker 的场景

- 你没有 VPS，但想要比机场更好的 IP 信誉
- 一般性的工作浏览和 API 调用，不需要 IP 固定
- 作为 VPS 节点不可用时的临时备用

### 免费额度说明

每天 10 万次请求，对正常工作用途基本够用。如果你在跑批量脚本，可能需要升级 Cloudflare Workers Paid（约 $5/月，每月 1000 万次请求）。

## 方式三：添加 VPS / SSH 节点

如果你需要**固定的独享 IP**，这是唯一正确的选择。

### 什么时候必须用 VPS

- **外贸收款账号**（Stripe、PayPal）：支付平台对 IP 变化极敏感，IP 突然变化会触发风控
- **广告账号**（Google Ads、Facebook Ads）：广告平台会关联 IP 和账号，IP 经常变化会被标记
- **平台多账号管理**（独立站、Amazon Seller、eBay）：平台通过 IP 关联多账号，需要稳定固定的 IP
- **OpenAI / Claude 账号长期稳定使用**：避免因 IP 变化触发反复验证

### 如何获取 VPS

推荐的海外 VPS 提供商（价格约 $3–8/月）：

- **Vultr**：按小时计费，可按需开关，节点选择灵活
- **DigitalOcean**：稳定性好，文档完善
- **Bandwagon Host（搬瓦工）**：国内用户常用，有针对中国优化的线路
- **Hetzner**：欧洲节点，价格便宜

购买时选择你业务需要的地区，例如：
- 美国节点：Google Ads、PayPal、OpenAI
- 日本/香港节点：延迟低，适合日常工作
- 德国节点：欧洲电商、GDPR 相关业务

### 在 VPS 上搭建代理服务

VPS 买好之后，需要在上面安装一个代理程序。常见方案：

**方案 A：使用 Shadowsocks / V2Ray（推荐新手）**

网上有大量一键脚本，例如：

```sh
# 在你的 VPS 上（SSH 登录后执行）
bash <(curl -fsSL https://raw.githubusercontent.com/xxx/xxx/install.sh)
```

搭建完成后，VPS 服务端会给你一个配置（服务器地址、端口、密码/UUID），把这个配置导入 ClashForge 即可。

**方案 B：使用 SSH 隧道（最简单，无需额外软件）**

如果你只是想快速测试，也可以直接用 SSH SOCKS5 代理：

```sh
ssh -D 1080 -N user@your-vps-ip
```

这会在本地开一个 SOCKS5 代理，把它填入 ClashForge 的节点配置里。

### 在 ClashForge 里添加 VPS 节点

在管理页面的节点管理里，选择"手动添加节点"，根据你的 VPS 上安装的代理协议（Shadowsocks、VMess、VLESS、Trojan 等）填入对应参数。

## 多来源并行配置

推荐配置：

| 设备 / 场景 | 使用哪个出口 |
| --- | --- |
| 工作主力电脑（外贸、AI 工具、广告） | VPS 固定节点 |
| 工作轻度需求（一般浏览、API） | Cloudflare Worker 节点 |
| 娱乐设备（手机、平板、电视） | 机场订阅 |
| 国内网站 / 服务 | 直连 |

在 ClashForge Web 页面的"设备规则"或"分流规则"里配置以上对应关系。

## 怎么知道配置成功了

| 检查项 | 期望结果 |
| --- | --- |
| 管理页面能打开 | `http://192.168.20.1:7777` 正常显示 |
| 能看到节点列表 | 对应节点已加载 |
| 代理服务已启动 | 状态显示"运行中" |
| 目标网站能访问 | 之前无法访问的网站现在能打开 |
| 出口 IP 符合预期 | 查询出口 IP，对应你选择的节点地区 |

遇到问题，先看 [不好用怎么办](/guide/troubleshooting)。
