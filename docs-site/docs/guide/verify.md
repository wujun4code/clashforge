# 怎么知道成功了

ClashForge 成功，不是指命令没有报错，而是指：**你的设备能稳定访问需要代理的资源，同时普通上网不受影响**。

## 最简单的判断

| 你看到的结果 | 说明 |
| --- | --- |
| 管理页面能打开 | ClashForge 服务在运行 |
| 节点列表能看到内容 | 订阅或配置已被读取 |
| 需要代理的网站能打开 | 代理服务基本可用 |
| 普通网站也能打开 | 没有明显影响正常上网 |
| 出口 IP 符合预期 | 设备确实通过代理访问 |

## 用命令检查

Windows：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 check
```

路由器本机：

```sh
clashforgectl check
```

这会做一次简单检查，帮你判断连接和出口是否正常。

## 用浏览器检查

用已经接入路由器的设备测试：

1. 打开一个平时需要代理才能访问的网站。
2. 打开一个平时不需要代理的国内网站。
3. 打开一个出口 IP 查询网站。

如果三项都符合预期，基本就可以继续日常使用。

## 什么情况算“不成功”

| 现象 | 可能情况 |
| --- | --- |
| 管理页面打不开 | ClashForge 没启动，或地址/端口不对 |
| 页面能打开但没有节点 | 订阅或配置没有导入成功 |
| 节点有，但网站打不开 | 代理节点不可用，或规则不符合预期 |
| 打开代理后普通网站也慢 | 配置或规则需要调整 |
| 所有设备都断网 | 先关闭代理相关设置，恢复网络 |

## 先恢复，再排查

如果已经影响上网，先执行：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 stop
```

恢复后再看 [不好用怎么办](/guide/troubleshooting)。

## 需要求助时

生成一份脱敏诊断报告：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.20.1 diag -Fetch -Redact
```

脱敏报告会尽量隐藏订阅链接、Token 等敏感信息。
