# 功能模块总览

这一页按模块解释 ClashForge 的核心功能。每个模块都回答四个问题：它是干嘛的，解决什么问题，如何使用，最终能带来什么价值。

## 1. 路由器管理控制台

**这个功能是干嘛的？**
ClashForge 把 mihomo 内核、配置文件、订阅、透明代理、防火墙、DNS 和诊断能力统一放到 OpenWrt 路由器上管理。Web UI 默认运行在 `http://<路由器IP>:7777`，后端提供 REST API 和 SSE 实时状态流。

**解决什么问题？**
传统做法是每台电脑、手机都装客户端，每个人自己维护订阅和节点。团队里只要有一个人配置错，就会出现 IP 混乱、节点失效、账号风控、难以排查的问题。路由器管理控制台把这些问题收敛到一个入口。

**如何使用？**
先按 [安装到路由器](/guide/install) 完成部署，再打开 Web UI。日常管理优先使用 Web 页面；需要恢复、升级或收集诊断时，使用 Windows/macOS/Linux 远程控制脚本，或在路由器本机临时执行 `clashforgectl.sh`。

**效果和价值**
所有连接到路由器的设备都可以被统一管理。你不需要挨个维护客户端，也更容易知道“当前谁在接管 DNS、谁在占用端口、哪个节点正在使用”。

## 2. 首次配置向导

**这个功能是干嘛的？**
配置向导把第一次上线拆成五步：导入配置、DNS 设置、透明代理设置、启动服务、连通检测。

**解决什么问题？**
直接把一份 Clash YAML 塞到路由器里很容易失败：端口冲突、DNS 冲突、GeoData 缺失、OpenClash 残留、规则不匹配都可能让网络异常。向导会在启动前预览最终配置，在启动后逐项验证端口和访问结果。

**如何使用？**
打开 Web UI，进入“代理服务”或“Setup”。先选择已有配置文件、订阅缓存、上传 YAML 或粘贴 YAML。然后按页面顺序确认 DNS 模式、透明代理模式和最终配置预览，最后点击“一键启动内核 + 应用接管”。

**效果和价值**
你能清楚看到 ClashForge 接管了哪些字段：端口、DNS、GeoData、设备分组规则。启动失败时也能看到具体卡在哪一步，而不是只看到“无法上网”。

## 3. 配置来源与订阅管理

**这个功能是干嘛的？**
ClashForge 支持保存多份 YAML 配置文件，添加多个订阅源，并把 SS、Trojan、VLESS、VMess 和 Clash YAML 统一解析成 mihomo 可用配置。

**解决什么问题？**
跨境团队通常会有主机场、备用机场、临时节点、自建节点和不同业务线的配置。如果这些配置散落在不同客户端里，更新、切换和排错都会很混乱。

**如何使用？**
进入“配置管理”。在“配置文件”里查看保存过的 YAML，在“订阅”里添加 URL、设置名称、User-Agent 和更新间隔。需要切换配置时点击“启动”，如果当前内核正在运行，ClashForge 会提示先停止再切换。

**效果和价值**
你可以保留多个来源并按需切换。订阅更新、节点数量、上次更新时间、运行中配置都可见，避免“我现在到底用的是哪份配置”这种典型排障难题。

## 4. 设备分流

**这个功能是干嘛的？**
设备分流允许你按源 IP/CIDR 给设备分组，并为每个设备组覆盖原始策略组的可选节点。例如“运营电脑 - 节点选择”只允许使用美国 VPS，“娱乐设备 - 节点选择”只允许使用机场节点。

**解决什么问题？**
同一个办公室里，销售、运营、开发、财务、访客设备的网络要求完全不同。全局开代理会让财务和国内系统变慢；所有人共用机场节点又会影响广告、支付和 AI 账号稳定性。

**如何使用？**
进入“设备分流”。选择一个配置来源，添加设备分组，从在线设备列表选择设备 IP，或手动填写 IP/CIDR。然后在“策略覆盖”里选择原始策略组，并勾选这个设备组允许使用的节点。保存后，ClashForge 会重新生成最终 mihomo 配置。

**效果和价值**
设备策略在路由器层生效，员工不需要理解代理规则。关键设备能长期走固定出口，普通设备仍按全局规则走，访客设备也可以保持隔离或直连。

::: warning 建议
给关键设备绑定 DHCP 静态租约。设备分流以源 IP/CIDR 命中，如果设备 IP 变化，规则也会失效。
:::

## 5. 出口节点管理

**这个功能是干嘛的？**
ClashForge 可以管理三类出口：机场订阅、自建 VPS/SSH 节点、Cloudflare Worker 节点。VPS 节点支持远程部署 GOST、Cloudflare DNS 绑定、Let's Encrypt 证书签发和 Clash 配置导出；Worker 节点支持部署 VLESS-WS 并导出 Clash YAML。

**解决什么问题？**
机场共享 IP 适合娱乐和普通浏览，但对广告账号、支付账号、AI 账号、平台后台并不稳定。你需要把“便宜共享出口”和“关键业务固定出口”放在一个地方管理。

**如何使用？**
进入“出口节点”。先配置 Cloudflare 凭据。需要 VPS 时点击“新增节点”，按向导填写服务器 IP、SSH 用户、域名和邮箱，完成授权、部署、证书和探测。需要 Worker 时点击“Worker 节点”，按向导部署并导出配置。

**效果和价值**
你可以把自建出口纳入同一个节点池，再通过设备分流和订阅发布使用。关键业务不再被机场节点质量牵着走。

## 6. 订阅定制与发布

**这个功能是干嘛的？**
订阅发布模块把已部署节点、内置模板、当前运行配置或自定义模板合并，发布到 Cloudflare Worker + KV，生成可直接给客户端使用的订阅链接。

**解决什么问题？**
团队里可能不只有 OpenWrt 路由器，还会有手机、笔记本、临时设备需要使用同一批自建节点。如果每台设备都手动复制节点配置，很快就不可维护。

**如何使用？**
进入“订阅定制”。先创建 Worker 托管环境，再选择模板模式：内置模板、当前运行配置或自定义模板。勾选要发布的节点，点击“预览最终 YAML”，确认后创建订阅链接。

**效果和价值**
你可以把自建节点以版本化订阅链接分发给团队设备。发布记录可复制、打开和删除，适合小团队内部维护一套统一配置。

## 7. 透明代理与 DNS 接管

**这个功能是干嘛的？**
ClashForge 根据配置应用 tproxy、redir、tun 或 none 模式，并管理 nftables/iptables 规则。DNS 侧支持 fake-ip / redir-host，并能与 dnsmasq 以 none、upstream 或 replace 方式共存。

**解决什么问题？**
路由器级代理最难的是“流量怎么进 mihomo”和“DNS 到底谁负责”。如果 DNS 直连、代理规则和透明代理不一致，就会出现国外网站打不开、国内网站变慢、设备全断网等问题。

**如何使用？**
首次使用建议在向导里保持默认值，先跑通一台设备。熟悉后再进入“高级管理”调整网络模式、DNS 模式、是否开机自动接管、是否绕过 LAN 和中国大陆 IP。

**效果和价值**
需要代理的流量能自动进入 mihomo，不需要每台设备配置代理端口。DNS 与规则保持一致后，分流结果也更可预测。

## 8. 规则集与 GeoData 管理

**这个功能是干嘛的？**
配置管理页可以查看 rule-provider、规则数量、文件大小，支持单个或全量同步，并能搜索某个域名或 IP 命中了哪些规则。GeoData 页面管理 `GeoIP.dat`、`GeoSite.dat` 或 mmdb 数据文件，支持手动和定时更新。

**解决什么问题？**
规则过期会导致分流错误，GeoData 缺失会导致 mihomo 启动或规则匹配异常。很多“为什么这个网站直连了”的问题，其实是规则集或 GeoData 的问题。

**如何使用？**
内核运行后进入“配置管理 -> 规则集”，查看和同步规则。在“路由数据”页面选择下载代理、点击立即更新，或开启定时更新。

**效果和价值**
你能知道规则文件是否存在、是否过期、目标域名到底命中了什么。排障不再只靠猜。

## 9. 连通性诊断与节点切换

**这个功能是干嘛的？**
概览页展示内核状态、活跃连接、实时上下行、资源占用、代理组、节点延迟、路由器侧检测、浏览器侧检测和域名探测。

**解决什么问题？**
很多故障需要区分：是路由器访问失败，还是当前浏览器直连失败；是 DNS 问题，还是节点问题；是目标网站屏蔽 IP，还是规则走错了。

**如何使用？**
进入“概览”。启动内核后先点“重新检测”，查看路由器侧和浏览器侧结果。切换 Selector 节点后，页面会自动重新检测。需要排查某个域名时使用“域名测试”。

**效果和价值**
你可以快速判断问题边界：节点不可用、DNS 失败、目标 IP 信誉差、还是本机网络异常。团队排障时也更容易描述现象。

## 10. 运维恢复与诊断

**这个功能是干嘛的？**
`clashforgectl.sh` 提供 status、compat、check、stop、reset、upgrade、diag、uninstall、openclash 等命令。Windows、macOS、Linux 远程脚本会把它临时上传到路由器执行；路由器本机也可以下载后直接执行。

**解决什么问题？**
路由器代理一旦配置错，影响的是全网。ClashForge 的运维命令把恢复网络放在第一位：先清理 DNS、nftables、策略路由和进程，再继续排查。

**如何使用？**
网络异常时优先执行：

```sh
sh /tmp/clashforgectl.sh stop
```

需要求助时生成脱敏诊断报告：

```sh
sh /tmp/clashforgectl.sh diag --redact
```

远程 Windows 示例：

```powershell
.\scripts\clashforgectl.ps1 -Router 192.168.1.1 diag -Fetch -Redact
```

**效果和价值**
即使配置出错，也有明确的退路。对没有专职网络工程师的小团队来说，这一点比“功能多”更重要。
